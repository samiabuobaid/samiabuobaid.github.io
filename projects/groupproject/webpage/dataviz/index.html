Data Visualizations

Our first graphic is a map of Allegheny County with all of the restaurants pulled from Yelp. We were able to first create the map with the longitude and latitude data which displayed each restaurant as a simple dot. The next step was to figure out which data could be used to enhance the map so that it would become a good tool for consumers. The first step was to include the zip code areas on the map to improve the relatability of where the restaurants are located. From there, we decided to group the restaurants by their price ranges in order to to tie in where you could eat at any price. The groupings are color coded, and each individual restaurant has five labels to provide greater insight into the map. Each restaurant has its’ name, street address, price, average rating, and review count to help the consumer explore their options.



For our second graphic, we decided to use a treemap in order to illustrate the different food categories and the frequency per zip code. While the size helps display the amount, there is also a heat map to help clarify the amount of each food genre. For the zip codes with many restaurants, it is a useful visualization to use to understand the diversity of the food scene in each area.



With our third graphic, we originally wanted to use a pie chart to show how many restaurants fit into each price category. The problem with a pie chart is that it groups all the restaurants together, but we wanted to create a more comprehensive visualization that would allow for the consumer to be able to view which restaurants fit where. In order to keep the original idea and spruce it up so that it would show more, we settled on packed bubbles. It shared the pie chart’s characteristic representation of each price group, while also containing three labels. The included labels are the restaurant name, address, as well as the price. Although the packed bubbles visualization already has the prices sorted by color, it allows for the consumer to be able to hover over any bubbles and find the relevant info without guessing.



For the final graphic, we had already used much of our data to create visualizations, so it took some time to create one that would be able to compliment the previous ones. We used the packed bubbles again to illustrate how many restaurants are on a specific street. Instead of using the previous graphics style to imitate a pie chart, we used the traditional use of size to display the data. The top five are shown as a way to give another tool to the consumer and also out of necessity. The top five were the largest and were able to fit the street name labels inside the bubbles. Although there are only five with labels, the option for the consumer to hover over the data points and find out what other streets had a large number of restaurants.


Alone, our graphics tell part of our story but not the whole. Each are to be used together to get a greater sense of the big picture. Hopefully the graphics will have been a quality insight for any consumer when trying to find a place to grub.
